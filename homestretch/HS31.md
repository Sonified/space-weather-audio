# HS31 — Spectrogram Shift on Speed Change

> **SHIP_IT.md:** Add spectrogram shift on speed change option in Data Playback panel on study builder

---

## Context: Existing Pipeline

The stretch pipeline is fully implemented in `js/audio-player.js` and `js/wavelet-gpu-compute.js`. The spectrogram renderer already has viewport/zoom controls via `main-window-renderer.js`.

### What It Means
When playback speed changes (e.g., 1.25x), the spectrogram's time axis should optionally compress/expand to match the audio timing. Currently, the spectrogram shows real-time data regardless of playback speed. The "shift" would adjust the time-to-pixel mapping by the speed factor.

### Key Files
- **`study-builder.html`**: Data Playback panel (line ~1754) — where the toggle goes
- **`js/audio-player.js`**: `updatePlaybackSpeed()` (line ~297) — where speed changes happen
- **`js/main-window-renderer.js`**: Viewport/zoom controls — where spectrogram scaling would be applied
- **`study-flow.js`**: Study config wiring

## Proposed Implementation

1. **Add toggle in study builder** Data Playback panel (~15 lines HTML):
   ```html
   <span class="toggle-label">Shift spectrogram on speed change</span>
   <div class="toggle toggle-spectrogram-shift ${...}" onclick="this.classList.toggle('on')"></div>
   ```

2. **Wire setting into study config** via `collectStudyConfig()` / `loadConfigIntoBuilder()`

3. **In `audio-player.js` or `main-window-renderer.js`**, when speed changes and flag is set, scale the spectrogram viewport by the speed factor

~1-2 hours. New toggle + viewport scaling logic.
