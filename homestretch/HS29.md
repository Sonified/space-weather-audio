# HS29 — Playhead Jump with Opacity Fade

> **SHIP_IT.md:** Handle playhead jumps with an opacity fade while the stretch catches up

---

## Context: Existing Pipeline

The stretch pipeline is fully implemented. `stretch_test.html` already has playhead tracking with source-time position math, looping, seek, pause/resume with proper position bookkeeping.

### What Already Exists
- **`stretch_test.html`**: Playhead tracking with source-time position math that handles stretch factor changes mid-playback. Seek/loop/pause all work.
- **`js/audio-player.js`**: `waveletStretchAndLoad()` handles re-stretch on speed change.

### What May Need Wiring
The fade effect exists in `stretch_test.html` but may need to be ported to the main app's spectrogram/waveform overlay. When the user seeks to a new position, the spectrogram area should briefly fade (opacity transition) while the stretch buffer catches up.

## Proposed Implementation

In the main app, when a playhead jump occurs during stretched playback:
1. Apply CSS opacity transition to the spectrogram container
2. Fade to ~0.5 opacity
3. When the new stretch buffer is ready, fade back to 1.0
4. If stretch is cached (see HS26-28), skip the fade entirely

## Complexity

~30 min. Partially done in stretch_test.html, may need wiring in main app.
