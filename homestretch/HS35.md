# HS35 — Study Builder "New Study" Flow

> **SHIP_IT.md:** Study builder: "New Study" flow — a button that presents three options: (1) Start Fresh (blank study), (2) Clone This Study (duplicate current config as a new study), (3) Begin From Template (pick from a list of saved templates). Consider what elegance looks like here — clean modal or inline dropdown, minimal friction.

---

## Architecture Context

### How Studies Are Identified
- **Slug** (URL tag): primary identifier via `nameToSlug()` (line 4757) — lowercase alphanumeric + hyphens, max 60 chars
- Used as: D1 primary key, IndexedDB keyPath, URL parameter

### Key Functions
- `collectStudyConfig()` (line 4264) — gathers all UI state into config object
- `loadConfigIntoBuilder(config)` — loads a config into the builder UI
- `saveDraft()` (line 4570) — saves to localStorage + IndexedDB + D1
- `saveAsToServer()` (line 2568) — already prompts for new name, changes slug, saves (does 90% of "Clone")
- `nameToSlug(name)` (line 4757) — generates slug from study name

### Study Config Structure
```js
{
  slug: "emic-pilot",
  name: "EMIC Wave Study",
  steps: [...],
  conditions: [...],
  randomization: {...},
  returningParticipant: {...},
  appSettings: {...},
  updatedAt: "ISO string"
}
```

## Proposed UI: Dropdown Menu

A new button in the header actions row:
```
[+ New Study ▾]
```

### Option 1: Start Fresh
- Clears all fields, generates blank config
- Prompts for study name (inline or small modal)
- Generates slug from name
- Calls `loadConfigIntoBuilder(blankConfig)`

### Option 2: Clone This Study
```js
function cloneStudy() {
  const config = collectStudyConfig();
  const clone = JSON.parse(JSON.stringify(config));
  const newName = prompt('New study name:', config.name + ' (copy)');
  if (!newName) return;
  clone.name = newName;
  clone.slug = nameToSlug(newName);
  clone.updatedAt = new Date().toISOString();
  // Participant data, sessions, snapshots are all keyed by slug in D1 —
  // won't exist for the new slug, so nothing to reset
  loadConfigIntoBuilder(clone);
  saveDraft();
}
```

Note: `saveAsToServer()` (line 2568) already does almost exactly this! The "Clone" option is essentially the same flow with better UX framing.

### Option 3: Begin From Template

**Template system (MVP — Option 1, simplest):** Templates are just study configs with `isTemplate: true` flag, stored in the same IndexedDB store. Zero backend changes.

- "Save as Template" action in File menu (adds `isTemplate: true, templateName, templateDescription`)
- Template listing function (filter IndexedDB by `isTemplate`)
- Template picker UI in the dropdown sub-menu
- ~100 lines JS, ~20 lines CSS
